pomolo 一些概念的介绍

  • gate服务器

    客户的发包后最早接受到的服务器，不参与rpc调用。故可以不需要por字段，只保存前段的port，主要作用为对前段做负载均衡，分配客户的前往的connector服务器。分配策略为根据客户的的key坐hash实现负载均衡。

  • connector服务器

    接受客户端的请求，主要为在客户端请求以及给客户端回复时的路由作用。

  • 应用逻辑服务器/后端服务器

    对实际的应用逻辑进行处理，可能有多个，之间进行rpc调用。

  • master服务器

    主要为读取配置文件，启动配置服务器的集群以及服务器的管理

  • rpc调用种类

    • 后端服务器向前端服务器请求session
    • 后端服务器通过channel推送消息对前段发起rpc
    • 前端将用户请求给后端

  • router

    用来标示具体的服务或者客户端接受服务 的位置。如服务端发送chat.chatHandler.send,而客户端则为onxxx协议，客户端请求消息后到达服务器经过一个路由函数router将其送到对应的服务器。

  • Session

    一个Session时一个客户端连接的抽象，相当于一个服务器客户端之间的connection，有唯一id和socket，一旦建立连接即不可以随便修改。其中，有一个FrountendSession概念

    而SessionService维护所有原始的Session信息。BackendSession与FrountendSession类似，不过其针对的是后端的连接。

  • channel

    channel是玩家id的容器，主要用于需要广播推送消息的时候使用。可以把玩家加入一个channel中，当需要对channle推送消息时，channel内的都会收到。channel是属于单个服务器本地的，并不能与其他服务器共享。

  • pomolo的四种消息类型

    request、response、notify（客户端给服务器）、push（服务器给客户端）

  • filter

    有before和after，每类filter可以注册多个，所有客户端的请求都会经过filter区处理，afterfilter主要控制处理之后的数据是否合法

  • handler/error handler

    处理业务逻辑的地方，在before filter，after filter之间。

    errorhander对流处理发生的异常进行集中的处理。

  • component

    pomelo中各个功能以组件模式构成

  • admin clinent,monitor,master

    monitor在各个应用服务器中，向master注册自己，上报服务器信息，同时接受master推送来的变化消息

    master负责收集整个服务器群的信息，有变化时推送到各个monitor，接受client发出的命令

    client独立运行进程，发起到master的连接，通过对master请求或命令来管理服务器群

  • admin module

    负责处理监控信息
